## 2. 项目目标

实现一个针对上海电子税务局出口退税业务的浏览器自动化程序，处理流程如下：

1. 打开网页：`https://etax.shanghai.chinatax.gov.cn:8443/#/`
2. 点击“登录”
3. 人工扫码登录
4. 登录成功后，程序继续导航：
   - 地方特色
   - 特色办税
   - 单一窗口出口退（免）税办理
   - 在线申报
   - 免退税申报
   - 逐项配单
5. “逐项配单”页面中，按 Excel 每一行顺序搜索“报关单号”
6. 找到完全匹配的网页行后点击“配单”
7. 点击“选择发票信息”
8. 根据 Excel 的“发票号码 + 备注1/备注2/备注3/备注4”匹配网页发票行号并勾选
9. 核对已选发票明细；一致则保存，不一致则按规则调整数量/计税金额后保存
10. 返回“逐项配单”页面，处理下一行
11. 一直循环到 Excel 最后一行结束

## 3. 当前已确认的业务规则

### 3.1 Excel 输入文件

固定文件：

`C:/Users/Administrator/Desktop/出口退税-图吉.xlsx`

当前已确认：
- 工作表：`Sheet2`
- 读取方式：按**中文表头**直接读取
- 处理方式：**严格按 Excel 行顺序逐行处理**
- 即使报关单号重复，也**不合并**，仍按每一行分别执行

### 3.2 Excel 已识别到的关键列

样表中已确认存在以下列：

- `货物品名`
- `成交数量`
- `成交单位`
- `计税金额`
- `发票号码`
- `发票号码行号`
- `报关单号`
- `报关单号项号`
- `备注`

另外样表中实际还出现了额外备注列位置（例如 U/V/W 列场景），程序设计上会把这些备注位统一解析为“可选发票行号”。

### 3.3 第一步网页行匹配规则

在“逐项配单”搜索结果中，只有当以下 5 个值**完全一致**时，才点击该行“配单”：

- 网页 `报关单号` = Excel `报关单号`
- 网页 `商品名称` = Excel `货物品名`
- 网页 `项号` = Excel `报关单号项号`
- 网页 `成交单位` = Excel `成交单位`
- 网页 `成交数量` = Excel `成交数量`

### 3.4 发票勾选规则

在“选择发票信息”弹窗中：

先满足：
- 网页 `发票号码` = Excel `发票号码`

再根据当前 Excel 行里的备注数字决定勾选哪些网页行：
- 如果 `备注1/备注2/备注3/备注4` 中存在数字
- 则勾选网页中 `发票行号` 等于这些数字的行
- 没有数字则不额外勾选

**当前实现约定**：
- 程序会把 `发票号码行号` 和备注中解析出的数字一起整理为候选发票行号集合
- 进入真实页面联调时，如果税务网页的实际业务关系与样表有差异，再做最后微调

### 3.5 保存前核对规则

保存前需要核对：

- 网页 `发票号码` = Excel `发票号码`
- 网页 `商品名称` = Excel `货物品名`
- 网页 `发票行号` = Excel `发票号码行号`
- 网页 `出口/进货数量` = Excel `成交数量`
- 网页 `发票单位` = Excel `成交单位`
- 网页 `计税金额` = Excel `计税金额`

### 3.6 自动修正规则

如果只有下面两项不一致：

- 网页 `出口/进货数量`
- 网页 `计税金额`

则允许程序自动修正：

- 找到网页端“出口/进货数量”最大的那一行
- 修改该行的数量和计税金额
- 使网页多行合计后满足：
  - 合计 `出口/进货数量` = Excel `成交数量`
  - 合计 `计税金额` = Excel `计税金额`

如果是其他字段不一致：
- 不自动猜测修复
- 直接停止并保留浏览器现场

### 3.7 异常处理策略

遇到以下情况时：
- 网页没找到匹配行
- 发票号码不匹配
- 发票行号勾选失败
- 保存失败
- 核对字段不一致且不属于允许自动修正范围

程序策略：
- **立即停止**
- **保留浏览器窗口**
- **写入日志和状态文件**
- 方便人工接管检查

## 4. 技术栈

本项目采用以下技术栈：

- **Node.js**
- **CommonJS**（`require` / `module.exports`）
- **Playwright**：浏览器自动化
- **Google Chrome**：实际驱动浏览器
- **xlsx**：读取本地 Excel 文件
- **本地 JSON 状态文件**：记录进度和失败信息
- **本地日志文件**：记录运行过程和错误现场

### 为什么选这套技术栈

因为它和你现有 `dxm` 程序的风格一致，适合这种：
- 需要人工先登录一次
- 后续程序长期操作网页
- 需要稳定复用登录态
- 需要按 Excel 批量驱动
- 需要出错时保留现场给人工查看

## 10. 运行方式与易操作方案

考虑到后续使用者不一定是专业开发人员，程序的运行、暂停、终止，建议采用**非常直观的双模式方案**：

### 10.1 推荐的操作方式

1. **探查模式**
   - 用于首次登录、菜单定位、页面结构检查
   - 不执行真实写入动作
   - 只打开网页、截图、保存页面结构信息

2. **正式运行模式**
   - 等网页结构确认后再启用
   - 才真正执行逐项配单、发票勾选、保存

### 10.2 适合非专业人员的控制方案

推荐后续做成这三种控制方式：

#### A. 双击启动 bat 文件
- `run_tax_refund.bat`
- 用户双击即可启动
- bat 里给出中文提示
- 不要求用户手敲命令

#### B. 键盘级控制
程序运行时支持：
- **回车**：继续下一步（例如扫码后继续）
- **Ctrl+C**：立即停止程序
- **关闭浏览器窗口**：视为人工接管，程序检测后停止

#### C. 文件开关控制（后续可加）
为了让“暂停 / 继续 / 停止”更容易，可以后续增加控制文件：
- `state/pause.txt`：存在时程序暂停轮询，等待恢复
- `state/stop.txt`：存在时程序安全停止

这样非专业人员甚至不需要碰终端：
- 想暂停：新建 `pause.txt`
- 想继续：删除 `pause.txt`
- 想终止：新建 `stop.txt`

这种方式很适合办公场景。

### 10.3 当前阶段先做什么
